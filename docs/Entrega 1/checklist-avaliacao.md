# Checklist de Avaliacao - Entrega 1

Use esta lista para conferir se a entrega cobre os pontos esperados no bloco de Concepcao.

## 1. Identificacao do grupo

- [x] Numero do grupo preenchido.
- [x] Nome completo e RA de todos os integrantes preenchidos.

## 2. Definicao do problema e modelo de negocio

- [x] Problema do projeto descrito com clareza.
- [x] Publico-alvo definido.
- [x] Contexto social/comunitario descrito.
- [x] Proposta de valor apresentada.
- [x] Business Model Canvas descrito em formato textual.

## 3. Requisitos do sistema

- [x] Minimo de 10 requisitos funcionais atendido.
- [x] Minimo de 5 requisitos nao funcionais atendido.
- [x] RNFs com justificativa.

## 4. User Stories e Kanban

- [x] Minimo de 10 user stories atendido.
- [x] Link do GitHub Projects/Kanban registrado.
- [x] Colunas esperadas do Kanban descritas no relatorio.

## 5. Repositorio GitHub

- [x] Link do repositorio publico registrado.
- [x] README.md raiz criado com nome do projeto, descricao, integrantes e instrucoes de uso.
- [x] Conferencia final de commits de todos os integrantes validada diretamente no GitHub.
